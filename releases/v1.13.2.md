# v1.13.2 — API Version Fix for Collections & Account Endpoints

## Bug Fixes

- **Fix `list_collections` 400 Bad Request** — `_make_request` was unconditionally
  overwriting `params["api-version"]` with the DataMap version (`2024-03-01-preview`),
  ignoring the correct Account API version (`2019-11-01-preview`) passed by callers.
  Now the caller-supplied version is preserved; only requests that omit `api-version`
  fall back to the DataMap default.  
  Affected operations: `list_collections`, `get_collection`, `create_collection`,
  `delete_collection`, `get_collection_path`, `get_account_properties`, and all other
  Account-plane endpoints.

## MCP Server

- Fixed stdio launch configuration for VS Code (`mcp.json`) — server now starts
  correctly using `python.exe -m purview_mcp_server.server` with
  `cwd: ${workspaceFolder}/tools/mcp`.

## Bug Fixes

### [FIXED] `pvw entity list` returning empty results

**Symptom:** Running `pvw entity list --limit N` returned `@search.count: 0` and an empty
`value` array even when the Purview catalog contained thousands of entities.

**Root cause:** The search payload passed `"keywords": "*"` to the Purview
`/datamap/api/search/query` API. The API treats `"*"` as a literal keyword search rather than
a wildcard, yielding zero matches.

**Fix:** Changed `"keywords"` to `None` (serialised as JSON `null`), which instructs the API
to return all indexed entities.

**Impact:** `pvw entity list` now correctly returns all entities (e.g. 44,478 in a typical
environment) and supports `--limit` and `--type-name` filtering as expected.

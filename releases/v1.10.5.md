## Overview

This patch release fixes two bugs in the `pvw uc term sync-classic` command and adds a new
`--delete-removed` flag for full bidirectional reconciliation between Unified Catalog domains
and classic Purview glossaries.

## Bug Fixes

### [FIXED] `--update-existing` silently failing for all existing terms

**Symptom:** Running `sync-classic --update-existing` reported `[X] Failed: '--termGuid'` for
every existing term and showed `Updated 0 / Failed N` in the summary.

**Root cause:** `glossaryUpdateTerm()` requires a `--termGuid` key in its args dict (used to
build the PUT endpoint URL), but the call omitted it entirely.

**Fix:** Added `"--termGuid": existing_guid` to the `glossaryUpdateTerm()` call in Step 4.

---

### [FIXED] Windows cp1252 UnicodeEncodeError on startup

**Symptom:** The command crashed immediately on Windows consoles with:

```
UnicodeEncodeError: 'charmap' codec can't encode character '\u2192'
```

**Root cause:** The header string `"Unified Catalog -> Classic Glossary Sync"` contained a
Unicode arrow character (`\u2192`) which is not encodable in the cp1252 code page used by
default Windows terminals.

**Fix:** Replaced `->` Unicode arrow with ASCII `->`.

---

## New Features

### `--delete-removed` flag

Adds optional deletion of classic glossary terms that no longer exist in the source UC domain.

```bash
# Full reconciliation: create missing + update existing + delete removed
pvw uc term sync-classic \
  --domain-id "abc-123" \
  --glossary-guid "gloss-guid" \
  --update-existing \
  --delete-removed

# Preview before applying
pvw uc term sync-classic \
  --domain-id "abc-123" \
  --glossary-guid "gloss-guid" \
  --update-existing \
  --delete-removed \
  --dry-run
```

- Opt-in only - existing workflows are unaffected.
- Works correctly with `--dry-run` (shows `Would delete:` lines).
- Summary table now includes a `Deleted` row.

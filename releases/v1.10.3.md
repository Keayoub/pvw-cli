# v1.10.3 Release Notes

## What's New

### Multi-value `--attr-value` support for Business Metadata commands

`pvw entity add-business-metadata` and `pvw entity add-business-metadata-attributes` now
accept JSON array literals in `--attr-value`, enabling SET/LIST typed attributes to be
set inline without a `--payload-file`.

**Before** — only plain strings were supported:

```powershell
pvw entity add-business-metadata-attributes `
  --guid <guid> --bm-name Metadata --attr-name Keywords `
  --attr-value "glossary"
```

**After** — JSON array literals are parsed automatically:

```powershell
pvw entity add-business-metadata-attributes `
  --guid <guid> --bm-name Metadata --attr-name Keywords `
  --attr-value '["Glossary Platform","Metadata Repository"]'
```

Resolution order for `--attr-value`:

1. JSON array/object literal (e.g. `["val1","val2"]`) - parsed to a list
2. Plain scalar string - passed as-is

This is consistent with the multi-value format used in CSV bulk import files
(e.g. `customAttributes.Metadata.Keywords` column).

None. Plain string values are unchanged.

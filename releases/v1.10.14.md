# Release v1.10.14

**Release Date:** April 23, 2026  
**Status:** Production Ready  
**Previous Release:** [v1.10.13](v1.10.13.md)

---

## Overview

Patch release focused on operational cleanup scenarios in Purview:

- Collection force-delete now includes lineage/process dependency detection.
- Self-hosted integration runtime deletion now includes scan dependency checks.
- Build verification in release flow now uses non-isolated package build mode for reliability.

---

## Key Improvements

### [NEW] Lineage-aware collection force-delete

`pvw collections force-delete` now inspects collection assets for linked lineage process entities
that can block deletion. It reports support-ready blocker GUIDs and can delete blockers with
relationship cleanup before collection deletion.

New options include:

- `--delete-lineage / --no-delete-lineage`
- `--lineage-depth`
- `--lineage-width`
- `--lineage-scan-limit`

### [NEW] Safe self-hosted integration runtime deletion

Added integration runtime commands under `pvw scan`, including a dependency-aware safe delete flow.

New commands:

- `pvw scan readintegrationruntimes`
- `pvw scan readintegrationruntime --integrationRuntimeName <name>`
- `pvw scan deleteintegrationruntime --integrationRuntimeName <name>`
- `pvw scan deleteintegrationruntimesafe --integrationRuntimeName <name> [--delete-dependent-scans] [--dry-run]`

Safe delete behavior:

- Discovers scans that reference the runtime via scan definitions.
- Lists all related scans and their run counts.
- Blocks deletion with clear guidance when dependent scans exist and auto-delete is not requested.
- Supports automatic dependent scan cleanup before runtime deletion.

### [IMPROVED] Release build verification reliability

Updated release build verification script to use:

- `python -m build --no-isolation`

This avoids environment-related stalls from isolated build bootstrap during release execution.

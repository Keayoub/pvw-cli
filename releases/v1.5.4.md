# v1.5.4 Release Notes

## Bug Fix: Windows Unicode Encoding Support

### Issue Fixed
**Problem**: CLI crashed on Windows when displaying Unicode symbols (✓, ⚠) due to CP1252 encoding limitations
**Error**: `UnicodeEncodeError: 'charmap' codec can't encode character '\u2713' in position 2: character maps to <undefined>`

### Solution
- **Fixed**: All Rich Console instances across the codebase now use `Console(legacy_windows=False)` 
- **Impact**: Enables proper Unicode rendering on Windows terminals
- **Scope**: 24 files updated (CLI commands, client modules, plugins)

### Files Updated

**CLI Modules:**
- `purviewcli/cli/unified_catalog.py`
- `purviewcli/cli/cli.py`
- `purviewcli/cli/domain.py`
- `purviewcli/cli/entity.py`
- `purviewcli/cli/health.py`

**Client Modules:**

**Plugin System:**
- `purviewcli/plugins/plugin_system.py`

### Technical Details

**Before:**
```python
console = Console()  # Used legacy Windows mode, failed on Unicode
```

**After:**
```python
console = Console(legacy_windows=False)  # Modern Windows terminal support
```

### Affected Commands
This fix resolves Unicode errors in all commands that display status symbols, including but not limited to:
- `pvw uc term import-csv`
- `pvw uc term update-csv`
- `pvw entity bulk-update`
- All commands using Rich console output with Unicode symbols

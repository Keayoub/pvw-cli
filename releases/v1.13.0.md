# Release Notes - v1.13.0

**Release Date**: May 22, 2026
**Previous Version**: 1.12.0

---

## Overview

v1.13.0 introduces a fully refactored MCP server hosted via Azure Functions, expanding coverage from 41 to **88 curated tools** across all Purview command groups.

---

## What's New

### MCP Server — Full Coverage Expansion (88 tools)

The Purview MCP server (`tools/mcp/`) has been rebuilt and now covers all major pvw-cli command groups:

- **Scan** (8 tools): list/get/create/delete data sources, run scans, scan history, rulesets
- **Insight** (8 tools): asset distribution, scan status, tags, data quality overview, lineage coverage, glossary usage
- **Account** (4 tools): account details, usage, limits, access keys
- **Workflow** (7 tools): list/get workflows, run history, approval requests, approve/reject
- **Policystore** (5 tools): data access policies, metadata policies, metadata roles, user permissions
- **Health** (2 tools): health summary, health actions
- **Share** (6 tools): sent/received shares, invitations
- **Data Quality** (7 tools): domains, connections, rules, quality score, scan execution
- **Unified Catalog** (12 tools): domains, terms, custom metadata, cleanup — unchanged from v1.12.0

All existing Atlas-layer tools (entity, glossary, lineage, search, types, collections, relationship) remain available.

### `invoke_operation` Escape Hatch — 18 Namespaces

All 9 new client namespaces (`account`, `insight`, `scan`, `management`, `policystore`, `workflow`, `share`, `health`, `quality`) are registered alongside the original 9, giving full programmatic access to 300+ pvw-cli methods via `invoke_operation` + `list_available_operations`.

### Azure Functions Hosting (`tools/hosting/`)

- New `tools/hosting/` directory with Azure Functions v2 (Python) app
- `_FastMCPBridge` ASGI wrapper pattern for FastMCP 2.x compatibility
- Environment variable documentation in `tools/hosting/README.md`
- `.vscode/mcp.json` wired to `http://localhost:7079/api/mcp`

### Old MCP Server Removed

The legacy `tools/PurviewMCPServer/` directory has been removed. All functionality is now in `tools/mcp/` and `tools/hosting/`.

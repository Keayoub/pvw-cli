# Release Notes - v1.14.3

**Release Date**: July 17, 2026
**Previous Version**: 1.14.2

---

## Overview

Enhancement and bug-fix release adding a new `pvw uc data-asset remove-relationship`
command, improving collection resource filtering, and hardening error handling for
relationship deletion in the Unified Catalog.

---

## New Commands

### `pvw uc data-asset remove-relationship`

Removes a relationship between a data asset and another entity (e.g. a Data Product).

```bash
pvw uc data-asset remove-relationship \
  --asset-id <ASSET_GUID> \
  --entity-id <PRODUCT_GUID>
```

The symmetric operation from the product side remains available via:

```bash
pvw uc dataproduct remove-relationship \
  --product-id <PRODUCT_GUID> \
  --entity-type DATAASSET \
  --entity-id <ASSET_GUID>
```

Use `--yes` to skip the confirmation prompt in automated workflows.

## Improvements

### `pvw collections resources` — enhanced asset-type filtering

Collection resource queries that combine `--asset-type` with `--collection-name`
now correctly construct an AND compound filter recognised by the Search API:

```json
{"and": [{"collectionId": "vdjesu"}, {"entityType": "fabric_lakehouse_table"}]}
```

Previously the flat filter caused the `collectionId` restriction to be silently
ignored, returning assets from unrelated collections.

### `pvw collections resources` — friendly-name resolution

`--collection-name` now resolves both the internal short-ID (e.g. `vdjesu`) and
the human-readable `friendlyName` (e.g. `"ws-dnai-p-ingestion"`).  An informative
warning is printed when neither form matches.

### `pvw uc dataproduct remove-relationship` — improved error handling

The CLI now correctly surfaces HTTP-level errors returned by the Unified Catalog
API, including a `PURVIEWCLI_DEBUG=1` hint for deeper diagnosis.

- **Enhanced `pvw uc term update-csv` command** to support multi-level custom attributes
- Custom attributes now support hierarchical structure using dot notation: `customAttributes.GroupName.AttributeName`
- Enables updating complex custom metadata structures in Purview Unified Catalog

**Example CSV:**
```csv
term_id,description,customAttributes.DataProductGovernance.BusinessDomain,customAttributes.DataProductGovernance.DataClassification
term-guid-1,Customer Data,Finance,PII
term-guid-2,Product Catalog,Product,Confidential
```

This creates the JSON structure:
```json
{
  "DataProductGovernance": {
    "BusinessDomain": "Finance",
    "DataClassification": "PII"
  }
}
```

## Migration Guide

Existing CSV files continue to work as before. To use nested attributes, simply add dots to your custom attribute column headers:

**Before (still works):**
```csv
customAttributes.Classification
```

**Now (supports nesting):**
```csv
customAttributes.DataProductGovernance.Classification
customAttributes.DataProductGovernance.Status
customAttributes.DataQuality.Score
```

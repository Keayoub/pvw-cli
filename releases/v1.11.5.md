# Release Notes - v1.11.5

**Release Date**: May 1, 2026  
**Previous Version**: 1.11.1

---

## Overview

v1.11.5 focuses on bulk entity reliability, lineage endpoint correctness, and release hardening.

This release also includes dependency remediation work and a documentation refresh that aligns command examples with the `pvw` entry point used throughout the CLI.

---

## Highlights

### 1. Bulk CSV update handling is more tolerant of real-world input

Bulk entity update workflows were improved to better handle CSVs that are not perfectly normalized.

Key improvements include:

- Header normalization for common CSV variations
- Better validation and error reporting when required identifiers are missing
- Safer parsing for flat attribute input during bulk processing
- More predictable behavior when updates rely on `guid` or `attributes.qualifiedName`

### 2. Classification bulk operations were refactored

Entity classification handling was cleaned up to better support bulk and schema-level workflows.

This includes:

- More consistent classification payload construction
- Better handling of multi-value classification input
- Cleaner compatibility behavior between CLI commands and client methods

### 3. Lineage endpoint handling was corrected

Lineage client routing was improved so GUID-based and unique-attribute lineage queries resolve against the proper endpoints with the expected query parameters.

This improves reliability for lineage reads and related follow-up analysis commands.

### 4. Security and release hygiene updates

The release train for 1.11.5 also included:

- Dependency updates to address reported vulnerabilities
- Regenerated lockfile output
- Security guidance and troubleshooting documentation
- Version alignment updates across packaging metadata

### 5. Documentation refresh

Documentation updates in the 1.11.5 range included:

- Modernized command examples using `pvw` instead of `python -m purviewcli`
- Navigation and visual styling improvements for the docs site
- Stronger release and security documentation coverage

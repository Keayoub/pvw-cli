# Release Notes - v1.11.8

**Release Date**: May 8, 2026
**Previous Version**: 1.11.7

---

## Overview

v1.11.8 is a tooling and packaging alignment release.

It consolidates all MCP server improvements, VS Code extension updates, and agent
customization work that accumulated alongside v1.11.7. No core CLI logic was changed.

---

## Highlights

### 1. MCP server version aligned to pvw-cli

The standalone MCP server package (`pvw-mcp-server`) and the npm launcher
(`chat.mcp.purview`) were brought into sync with the main pvw-cli version track.

Updated files:

- `tools/PurviewMCPServer/__version__.py` - version aligned to 1.11.6/1.11.7 baseline
- `tools/PurviewMCPServer/pyproject.toml` - dependency floor updated to `pvw-cli>=1.11.6`
- `tools/PurviewMCPServer/package.json` - npm package version aligned

### 2. MCP server bundled copy updated (VS Code extension)

The bundled server used by the VS Code extension was refreshed to match the standalone
server, including:

- `delete_entity` regression fix (was incorrectly calling `get_account_properties`)
- Dynamic operation registry (`_build_operation_catalog`, `_resolve_namespace_client`)
- `list_available_operations()` and `invoke_operation()` tools added

### 3. MCP server and extension documentation updated

Both README files were updated with:

- Install instructions for `pip`, `uv`, `uvx`, and `npx`
- MCP config JSON examples
- Advanced launch options section for the VS Code extension
- Markdown style normalization pass (no diagnostics)

### 4. Agent customization and .gitignore hygiene

- `.github/agents/PurviewCli.agent.md` updated with current agent profile, performance
  optimization patterns, and release workflow guidance
- `.gitignore` updated to exclude additional build and tool artifacts

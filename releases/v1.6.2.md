## üéØ Summary
Complete alignment of Collections API implementation with Microsoft Purview official specification. This release fixes all docstring inconsistencies and ensures API conformance.

## ‚úÖ Changes

### Collections API Compliance
- **Fixed Collections API Response Fields**: Updated all docstrings to reflect actual Microsoft Purview API response structure
  - Corrected field references from `guid` to `name`
  - Removed invalid fields: `attributes`, `status`, `updateTime`
  - Added proper fields: `friendlyName`, `description`, `parentCollection`, `collectionProvisioningState`

### Docstring Updates
- `collectionsRead()`: Shows correct response fields for single collection and list operations
- `collectionsCreate()`: Accurate response structure documentation
- `collectionsUpdate()`: Fixed example outputs to use `result['name']`
- `collectionsDelete()`: Documents 204 No Content response
- `collectionsReadPath()`: Shows `parentNameChain` and `parentFriendlyNameChain` structure
- `collectionsReadChildNames()`: Displays paginated list structure with `count`, `nextLink`, `value`
- `collectionsReadPermissions()`: Fixed docstring for permission queries
- `collectionsUpdatePermissions()`: Fixed docstring for permission updates
- `collectionsReadAnalytics()`: Proper parameter and response documentation
- `collectionsReadHierarchy()`: Hierarchical structure documentation
- `collectionsReadTree()`: Tree structure documentation

### API Version Consistency
- **20+ instances fixed**: All `get_api_version_params("account")` calls in Collections operations now use `get_api_version_params("collections")`
- Ensures explicit use of Collections API version `2019-11-01-preview`

### Endpoints Verification
- ‚úÖ Verified all endpoints match Microsoft Purview spec:
  - List: `GET /account/collections`
  - Get: `GET /account/collections/{collectionName}`
  - Create/Update: `PUT /account/collections/{collectionName}`
  - Delete: `DELETE /account/collections/{collectionName}`
  - Get Path: `POST /account/collections/{collectionName}/getCollectionPath`
  - List Children: `POST /account/collections/{collectionName}/getChildCollectionNames`

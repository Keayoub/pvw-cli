# Release Notes - v1.11.9

**Release Date**: May 11, 2026
**Previous Version**: 1.11.8

---

## Overview

v1.11.9 adds **bulk relationship automation** capabilities to the CLI with a new `bulk-create-csv` command for simplified management of relationships between objects of information (Commercial Assets) and data entities in Microsoft Purview.

---

## Highlights

### 1. New `pvw relationship bulk-create-csv` Command ⭐

A new streamlined CLI command for creating multiple relationships from a CSV file:

```bash
pvw relationship bulk-create-csv --csv-file entities_mapping.csv
```

**Features:**
- Read relationship definitions from CSV files
- Automatic validation of required columns
- Preview mode with `--dry-run` flag
- Optional JSON output with `--output-json`
- Color-coded status messages with emojis
- Comprehensive error handling and reporting
- Summary statistics on completion

**CSV Format:**
```csv
info_object_guid,target_entity_guid,target_entity_type,relationship_type
d286692e-30bb-48ba-ac49-f7372b12d225,90d14acb-cf75-4729-9245-68f6f6f60000,mssql_table,Objet Information_Table_Has
```

## Use Cases

### Scenario 1: Bulk Link Object of Information to Tables
```bash
# Preview the relationships
pvw relationship bulk-create-csv --csv-file mapping.csv --dry-run

# Create the relationships
pvw relationship bulk-create-csv --csv-file mapping.csv
```

```bash
# Create and save the JSON for future reference
pvw relationship bulk-create-csv --csv-file mapping.csv --output-json relationships.json
```

### Scenario 3: Automate with Scripts
```bash
# Use in automation pipelines
pvw relationship bulk-create-csv --csv-file entities_mapping.csv --dry-run
```

## Technical Details

### Command Signature
```bash
pvw relationship bulk-create-csv --csv-file <path> [--output-json <path>] [--dry-run]
```

### Options
| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `--csv-file` | Path | Yes | Path to CSV file with relationship mappings |
| `--output-json` | Path | No | Optional path to save generated JSON |
| `--dry-run` | Flag | No | Preview without creating (default: false) |

### Required CSV Columns
- `info_object_guid` - GUID of the Information Object
- `target_entity_guid` - GUID of the target entity
- `target_entity_type` - Type of target entity (e.g., mssql_table, mssql_procedure)
- `relationship_type` - Type of relationship (e.g., Objet Information_Table_Has)

## Performance Notes

The command:
- Reads CSV file once at startup
- Validates structure before any API calls
- Creates relationships sequentially with individual status updates
- Provides real-time feedback during batch operations
- Supports any CSV size (limited only by API rate limits)

For large-scale operations (100+ relationships), consider:
- Splitting CSV into batches
- Adding delays between API calls
- Using the advanced Python scripting approach documented in the guide

## Migration Guide

For users previously using manual relationship creation:

**Before (Manual):**
```bash
pvw relationship create --payload-file rel1.json
pvw relationship create --payload-file rel2.json
pvw relationship create --payload-file rel3.json
```

**After (Automated):**
```bash
# Create CSV with all relationships
# Then one command:
pvw relationship bulk-create-csv --csv-file all_relationships.csv
```

**Or for existing JSON files:**
```bash
# Still works exactly as before
pvw relationship create --payload-file relationship.json
```

## How to Report Issues

If you encounter any issues with the new `bulk-create-csv` command:

1. Run with `--dry-run` to isolate the problem
2. Check the CSV format matches the required columns
3. Verify GUIDs are valid and exist in Purview
4. File an issue on [GitHub](https://github.com/Keayoub/pvw-cli/issues) with:
   - Command executed
   - CSV sample (with sanitized GUIDs)
   - Error message
   - Purview account region

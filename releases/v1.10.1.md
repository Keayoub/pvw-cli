# Release v1.10.1 - CLI Invocation Cleanup and CSV Update Fixes

**Release Date:** March 11, 2026  
**Status:** Production Ready  
**Previous Release:** [v1.10.0](v1.10.0.md) (March 6, 2026)

---

## Overview

v1.10.1 is a patch release focused on command usability and bulk update reliability.

This release standardizes user-facing command examples on the `pvw` executable instead of `py -m purviewcli`, and it fixes common CSV parsing issues in `pvw uc term update-csv` that caused false `Missing term_id` errors.

---

## Included Changes

### 1. Standardized CLI invocation to `pvw`

User-facing examples, quick-start material, notebooks, and PowerShell samples now consistently use:

```bash
pvw <group> <command>
```

instead of legacy module-style invocations such as:

```bash
py -m purviewcli <group> <command>
```

Updated areas include:

- `README.md`
- `quick_start_guide.bat`
- Unified Catalog documentation
- Business metadata documentation and samples
- PowerShell examples
- Notebook examples

### 2. Improved `uc term update-csv` header parsing

`pvw uc term update-csv` now handles CSV files more robustly by supporting:

- UTF-8 with BOM (`utf-8-sig`)
- Automatic delimiter detection for `,`, `;`, tab, and `|`
- Flexible term ID header names

Accepted header variants now include:

- `term_id`
- `id`
- `ID`
- `Term ID`
- `TermId`
- `termId`
- `term-id`
- `term id`
- `termGuid`

This fixes a common case where valid CSV files were loaded, but every row was skipped with:

```text
Skipping row X: Missing term_id
```

### 3. Better diagnostics for invalid CSV headers

When no supported term ID column is present, the command now fails early with a clearer message showing:

- the expected header names
- the headers actually detected in the CSV
- the detected delimiter in `--debug` mode

---

## Impact

- Bulk term updates are now more tolerant of real-world CSV exports.
- Documentation and examples align with the installed CLI entry point: `pvw`.
- Troubleshooting bad CSV files is faster with clearer diagnostics.

---

## Upgrade

```bash
pip install --upgrade pvw-cli
```

Or install from source:

```bash
pip install dist/pvw_cli-1.10.1-py3-none-any.whl
```

---

## Notes

- No breaking changes are introduced in this patch release.
- Existing `python -m purviewcli` usage remains a legacy entry path, but `pvw` is now the recommended command in documentation.

# Release v1.10.25

**Release Date:** April 27, 2026  
**Status:** Production Ready  
**Previous Release:** [v1.10.14](v1.10.14.md)

---

## Overview

v1.10.25 introduces Unified Catalog Data Quality command support and tenant-aligned API routing.

This release adds a new `pvw uc quality` command surface, wires a dedicated Data Quality client,
and updates endpoint/version handling so domain-scoped quality operations work with current tenant behavior.

---

## Included Changes

### 1. New Unified Catalog Data Quality command group

Added canonical Data Quality commands under:

```bash
pvw uc quality
```

Supported command set includes:

- `pvw uc quality domains`
- `pvw uc quality domain-report --domain-id <DOMAIN_ID>`
- `pvw uc quality data-sources --domain-id <DOMAIN_ID>`
- `pvw uc quality schedules --domain-id <DOMAIN_ID>`
- `pvw uc quality alerts --domain-id <DOMAIN_ID>`
- `pvw uc quality assets --domain-id <DOMAIN_ID>`
- `pvw uc quality score domain --domain-id <DOMAIN_ID>`

### 2. New Data Quality client implementation

Added a dedicated Data Quality client to map CLI arguments to REST requests and support domain-scoped quality workflows.

Key additions:

- New client module for quality endpoints
- Request state reset behavior for cached client reuse
- Consistent paging/filter argument support (`--skip`, `--top`, `--filter`)
- Structured unsupported-operation responses where endpoints are not tenant-exposed

### 3. Endpoint and API version updates for quality namespace

Added `data_quality` endpoint mappings and a `quality` API version category:

- Quality namespace mapped under `/datagovernance/quality/...`
- Preview API version wiring for quality category
- Endpoint category mapping so quality calls resolve to the correct API version group

### 4. Unified Catalog quality routing and account resolution improvements

Updated sync client behavior to improve reliability for Unified Catalog quality calls:

- Included `/datagovernance/quality` in Unified Catalog routing path detection
- Improved Purview account ID discovery fallback logic
- Added fallback behavior using tenant/account discovery inputs when needed

### 5. Command tree placement finalized

Quality commands are registered under Unified Catalog and not exposed as a separate top-level command,
ensuring a single canonical path:

```bash
pvw uc quality
```
